# ADR-005: Scene-Family Minimum Contracts

**Status:** Proposed

## Context
MapScene is the priority, but BattleScene, MenuScene, and UI must not be afterthoughts.

## Decision
We have established a **Scene-Family Minimum Contract** that all translators (Map, Battle, Menu, Dialogue) must fulfill.

### Minimum Contract Specifications
| Scene Family | Contract Marker | Translation Requirement |
| :--- | :--- | :--- |
| **MapScene** | `SpatialMapOverlay` | Elevation coherence, Actor anchoring, Chunk-streaming support. |
| **BattleScene** | `FormationAnchoring` | Unit placement (Heroes/Enemies), Readability Guarantee (Contrast). |
| **MenuScene** | `LayeredComposition` | GPU budget < 1ms, Tier-aware Background Layering. |
| **DialogueScene** | `VisualEmphasis` | Focus blurring, Text-legibility override, Advanced Post-FX. |

### Minimum Feature Table:
All scenes, regardless of family, must satisfy the `FrameIntent` interface:
1.  **Actor List**: Must provide a list of entities and their `CapabilityTier`-aware visual models.
2.  **Visual Overrides**: Current environmental state (Fog, Lights).
3.  **Readability Tokens**: Requests for UI-space desaturation or blur.

## Rationale
Standardizing the `FrameIntent` ensures that the Render Backend remains agnostic to whether it is rendering a world map or a combat arena, while also providing hooks for scene-specific "Visual Readability" overrides.

## Consequences
- Contracts must be met before any scene adapter is considered shippable.
- Integration tests will be written against these contracts.
