# ADR-010: Memory and Allocation Model

**Status:** Proposed

## Context
Frame-by-frame allocation must not cause performance regressions or spikes.

## Decision
[TBD — define allocation strategy]
- Use of per-frame Arenas for `PresentationFrameIntent`.
- Heap-only for stable authoring data.

## Rationale
Zero-alloc steady state for the hot path.

## Consequences
- No `new` / `malloc` in the frame generation loop.
- Arena size must be bounded per tier.
