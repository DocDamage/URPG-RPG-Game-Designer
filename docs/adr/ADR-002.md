# ADR-002: Project Presentation Mode Model

**Status:** Proposed

## Context
The project needs to support toggleable modes (Classic 2D, Spatial) but some scenes (e.g. battles or intros) might need overrides or specific constraints.

## Decision
We have implemented a **Cascading Mode Model** for project-wide vs. scene-specific presentation settings.

### Mode Scoping Model
1.  **Project Level**: The `ProjectPresentationSettings` schema defines the baseline `CapabilityTier` and default `InterpretationMode` (e.g., Spatial-First).
2.  **Scene Level**: Individual scenes (Map, Battle) can declare a `MaterialResponseProfile` and `CameraProfile` to override the global defaults.
3.  **Actor/Entity Level**: Specific entities can override their individual `ActorPresentationProfile` (e.g., a "Classic 2D Sprite" actor in a "3D Spatial" map).

### Resolution Strategy
`Entity Override` > `Scene Override` > `Project Default`.

## Rationale
This hierarchical model enables "Hybrid Scenes"—such as a 2D menu overlaying a 3D battle—without breaking the engine's interpretation consistency. It also allows for individual actor fidelity scaling based on the detected `CapabilityTier`.

## Consequences
- Affects schema design for project settings and scene data.
- Influences editor UI for project-wide vs scene-level mode selection.
